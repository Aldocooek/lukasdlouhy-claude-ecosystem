## 1. Persuasion Psychology Stack

Core frameworks live in `sales-psychology-frameworks.md`. Additions below.

### Extended Buyer Psychology (Adam)

| Bias / Effect | Marketing Application |
|---|---|
| Mere Exposure Effect | Retargeting works because familiarity = trust; 7+ impressions before conversion |
| Endowment Effect | Free trials create ownership; loss aversion kicks in at cancel moment |
| IKEA Effect | Users overvalue what they helped configure — involve buyer in setup |
| Zero-Price Effect | "Free" triggers disproportionate response vs "$0.01" — use "free" explicitly |
| Hyperbolic Discounting | Near-term gains beat future ones; lead with immediate win, not long-term ROI |
| Goal-Gradient Effect | Show progress bars; people accelerate as they approach completion |
| Peak-End Rule | The worst moment + the last moment define the memory — make the end remarkable |
| Zeigarnik Effect | Incomplete loops demand closure — open loops in hooks, close in the offer |
| Pratfall Effect | Small imperfections increase credibility; admitting a minor flaw boosts trust |
| Paradox of Choice | >3 options stall decisions; default to 3 tiers, pre-select the middle |
| Mimetic Desire | People want what others they admire want — show aspirational users first |
| Bandwagon Effect | Specificity makes social proof work: "12,400 teams" beats "thousands of teams" |
| Regret Aversion | "Don't miss this" framing activates regret more than "get this" framing |

### Fogg Behavior Model
`Behavior = Motivation × Ability × Trigger`
- High motivation + low ability → simplify the action
- Low motivation + high ability → increase trigger salience (urgency, scarcity)
- All three required simultaneously for behavior to occur

### BJ Fogg EAST Framework
Make the desired action Easy, Attractive, Social, Timely.

## 2. Copy Frameworks Library

Full AIDA, PAS, BAB, 4U, Halbert, Hormozi in `sales-psychology-frameworks.md`. Below: additions and platform application.

### Hero Copy Formula (landing pages)
```
Headline:  [Specific outcome] for [specific audience] [without/in/by timeframe]
Sub:       One sentence resolving the headline's key ambiguity
Social bar: ★★★★★ X.X from N reviews + 4-6 logos
CTA:       Action + outcome ("Get the [name] — free")
```
Benefit-led headlines with a number outperform vague taglines by ~27% (CXL data).

### Fascination Bullet Templates (Halbert — extended)
1. "The [number] things to NEVER do when [situation]"
2. "How [unusual subject] [achieves surprising result]"
3. "What [expert type] does that [common audience] ignores"
4. "The [adjective] secret behind [credible result]"
5. "[Number]-second trick for [specific outcome]"
Rule: bullets promise outcomes, never reveal the mechanism. Leave the gap open.

### Direct Response Copy Sequence
Story (System 1) → Data (System 2) → Offer → Guarantee → Scarcity → CTA
PS line: restate offer + scarcity + social proof in one sentence. PS is the 2nd most-read element.

### Web Copy Principles (Adam)
- Clarity over cleverness — always
- Customer language, not internal jargon (use their exact words from reviews/interviews)
- Benefits > features, but name the mechanism — "how it works" beats "what it does"
- Specific beats vague: "Cut reporting time 75%" beats "faster reporting"
- Deliver 2-3 alternatives for every headline; the first draft is never the best

## 3. Cold Outreach Methodology

5-email sequence structure and deliverability gates are in `oneflow-and-raising.md`. Additions:

### Signal-Based vs. Quota-Based
Never send to a static list. Triggers that should fire outreach:
- New IČO / company registration
- Founder exit or M&A completion
- LinkedIn activity: saved content in your niche, commented on competitor post
- Pricing page visit, calculator use, content download (Tier 1 signal)
- Conference attendee list in your space

### Subject Line Formula
- 2-4 words, lowercase, no punctuation tricks
- Resembles internal email: "quick question", "intro from [name]", "[company] + [topic]"
- Avoid: "Following up", "Just checking in", "Exciting opportunity"

### Email Structure Options (Adam framework)
Three interchangeable frameworks — pick by context:
1. `Observation → Problem → Proof → Ask` (cold, no prior engagement)
2. `Question → Value → Ask` (warm; they've interacted with content)
3. `Trigger → Insight → Ask` (signal-based; use the trigger as opener)

### CTA Calibration
- Touch 1–2: preference-based ask — "Worth 15 minutes?" (not "Book a demo here")
- Touch 3: direct meeting request with specific slot
- Touch 4: single yes/no question, no pitch
- Touch 5: breakup email — "I'll stop here. Door's open if timing changes."

### Cold Email Hard Rules (merged Filip + Adam)
| Metric | Green | Yellow | HALT |
|---|---|---|---|
| Bounce rate | <2% | 2–4% | >4% |
| Spam rate | <0.1% | 0.1–0.3% | >0.3% |
| Open rate | >40% | 20–40% | <20% |
| Reply rate | >5% | 2–5% | <2% |
| Domain warmup | Day 21+ | Day 14–21 | <14 days |

### LinkedIn Outreach Benchmarks
- Connection acceptance: 20–35% with personalized note
- Reply on accepted connection: 5–8%
- Daily connection cap: 30 (humanized, not bot-uniform)
- A/B test message variants: 4 versions minimum before declaring a winner
- Sequence: Connect → Thank (no pitch) → Value×2 → Meeting ask

### WhatsApp [CZ-specific]
- Reply rate 26%+ on warm contacts (OneFlow 2026-04-22 benchmark)
- Max 80 words; open with their name + observation
- Only after LinkedIn or email contact established

## 4. Ad Creative Principles

### Hook Architecture (2-second retention rule)
The first 2–3 seconds determine whether someone keeps watching or scrolling. Hook must do exactly one of:
- Emotional contrast: "[negative fact] + [positive metric]"
- Pattern interrupt: violates genre expectation (unusual visual, counterintuitive claim)
- Relatable pain: "You're leaving [specific amount] on the table"
- Curiosity gap: names the payoff without revealing the mechanism
- Urgency command: "Save this — you'll need it"

Do NOT combine two hook types. Pick one.

### Ad Platform Character Limits
| Platform | Headline | Primary text visible | Max |
|---|---|---|---|
| Meta Ads | — | 125 chars before "more" | 600 chars total |
| Google RSA | 30 chars each | — | 15 headlines, 4 descriptions |
| LinkedIn | — | 150 chars recommended | 600 chars intro |
| TikTok / X | — | 80-100 chars | — |

Google RSA rule: every headline must make sense in isolation AND in any combination.

### Creative Production Workflow
1. Define 3-5 distinct angles: pain, outcome, social proof, curiosity, authority
2. Write 3+ variations per angle (adjust word choice, specificity, tone)
3. Validate against platform character limits
4. Ship angle-based test batches; kill losing angles after statistical significance (95%)
5. Double down on winners; introduce 1-2 new angles per cycle while protecting proven ones

### Instagram Algorithm 2026
Priority signals (highest to lowest): Shares → Watch time → Saves → Comments → Likes → Profile visits → Reach
Content that "brings people together" (DMs, shares) outranks passive-view content.
Create "I need to send this" content — the share is the metric.

## 5. Landing Page Conversion Patterns

Full 2026 patterns in `skills/landing-patterns-2026/SKILL.md`. Summary:

### Section Order (default)
1. Hero: benefit-led headline + number, one CTA, sub-headline, visual anchor
2. Social proof bar (below fold line): rating + N reviews + logos (+8-15% hero conversion)
3. Bento features grid (67% of top SaaS use this)
4. Mechanism explanation (sticky scroll or 3-step visual)
5. Pricing (3 tiers, middle pre-selected, annual default if >15% discount)
6. Testimonials (2-3 long-form with name, photo, specific result)
7. FAQ (5-7 awkward objections — price, switching cost, refund)
8. Final CTA (repeat hero)

### Conversion Rate Benchmarks
- Warm traffic lead capture: 20–40%
- Cold traffic lead capture: 5–15%
- Each extra form field: -5 to -10% completion
- Email-only capture converts highest; phone field is the #1 drop-off trigger

### Above-the-Fold Anti-Patterns
- Vague taglines ("Reimagine your workflow") — always add a number
- 3+ CTAs competing on same page
- Hero carousel (kills message clarity)
- Autoplaying audio (instant back)
- Stock photos (kills trust; use product screenshots or founder video)

## 6. Email Sequences & Deliverability

Full 5-email cold outreach sequence in `oneflow-and-raising.md`. Below: nurture + deliverability.

### Nurture Sequence Types
| Type | Emails | Duration | Goal |
|---|---|---|---|
| Welcome | 5-7 | 12-14 days | Activate + build trust |
| Lead nurture | 6-8 | 2-3 weeks | Overcome objections |
| Re-engagement | 3-4 | 2 weeks | Win back inactive |
| Onboarding | 5-7 | 14 days | Feature adoption |

### Welcome Sequence Structure (6-email, generalized from Filip/Adam)
| # | Day | Content | CTA |
|---|---|---|---|
| 1 | 0 | Who you are + what they can expect | "Read [story]" |
| 2 | 3 | Education: mechanism behind what you do | "Download guide" |
| 3 | 7 | Social proof: specific results + numbers | "See results" |
| 4 | 10 | Objections: risk, timeline, complexity | "FAQ" |
| 5 | 14 | Soft CTA: tool, calculator, consultation | "Calculate [outcome]" |
| 6 | 21 | Hard CTA: capacity limit or real deadline | "Reserve spot" |

80/20 rule: 80% value, 20% pitch. One email = one topic + one CTA.

### Deliverability Technical Gates (mandatory before sending)
- SPF + DKIM + DMARC configured on ALL outreach domains
- 21-day warmup minimum; max 50 emails/domain/day during warmup
- Three-domain rotation: A (priority) / B (testing) / C (warmup only)
- Proofpoint PDR check before any new domain goes live
- Bounce >4% OR spam >0.3% → HALT, do not send until resolved
- GDPR: unsubscribe link mandatory; opt-in must be documented

### Subject Line Best Practice
- 40-60 characters, 2-4 words for cold
- Patterns that work: questions, how-tos, numbers, direct statements
- Test: does it look like an email from a colleague?

## 7. Funnel Architecture

### TOFU / MOFU / BOFU Content Mapping
| Stage | Awareness Level (Schwartz) | Content Type | CTA |
|---|---|---|---|
| TOFU | Unaware / Problem-aware | Provocative data, stories, short-form video | Subscribe, download |
| MOFU | Solution-aware | Comparison, case study, webinar, demo | Trial, consultation |
| BOFU | Product-aware / Most aware | Testimonial, guarantee, urgency/scarcity | Buy, reserve, sign |

### Traffic Temperature Matching (from `marketing-funnel-audit` skill)
Cold traffic to a high-commitment page = conversion killer. Diagnose mismatch:
- TOFU mismatch: credit card asked before trust established
- MOFU mismatch: re-explaining basics to warm prospects
- BOFU mismatch: competing CTAs, no urgency, missing guarantee

### Retargeting Ladder
1. Content viewers → lead magnet offer (TOFU→MOFU bridge)
2. Lead magnet openers → case study or demo (MOFU)
3. Sales page visitors (no convert) → objection-handling content (MOFU→BOFU)
4. Cart/form abandoners → scarcity + guarantee reminder (BOFU)
5. Past buyers → upsell or referral ask

### A/B Testing Protocol (PIE Framework)
Prioritize tests by: **P**otential × **I**mportance × **E**ase
- Minimum test window: 2 weeks OR until 95% statistical significance
- Test one variable at a time — headline only, or CTA only, never both
- Kill losers fast; double-down on winners before testing new variations

## 8. Pricing & Monetization Psychology

Core anchoring, decoy, Hormozi in `sales-psychology-frameworks.md`. Additions:

### Charm Pricing vs. Rounded Pricing
- Charm pricing ($97, $197): signals "deal" — use for consumer/impulse purchases
- Round pricing ($100, $500): signals quality/premium — use for B2B and high-ticket
- Rule of 100: discounts <$100 → express as %; discounts >$100 → express as absolute amount

### Price Relativity (Mental Accounting)
Compare to an alternative cost, not to zero:
- "Less than a daily coffee" → trivializes the purchase
- "Less than one bad hire" → reframes against a real cost they've felt
- "Cover the cost of X with one additional [unit]" → ties to outcome

### Offer Stacking Structure
Name every component and assign it a value:
```
Core product:       $X
Bonus 1 (named):    Value $Y
Bonus 2 (named):    Value $Z
Total value:        $X+Y+Z
Today's price:      $[real price]
```
Bonuses must be adjacent to the core offer and genuinely useful — irrelevant bonuses backfire.

### Pricing Page Rules
- 3 tiers maximum; middle pre-selected as "Most Popular" or "Recommended"
- Annual default if discount >15% — monthly option present but not default
- Per-tier: price, 3-5 bullet benefits (not 15), tier-matched CTA text
- Show logos or testimonials on the pricing page — trust at the moment of decision

## 9. SEO & Programmatic Content

### E-E-A-T Framework (Google + AI Search)
| Pillar | Signals |
|---|---|
| Experience | Original research, case studies, personal documentation, unique data |
| Expertise | Author credentials, technical depth, accurate claims |
| Authoritativeness | External citations, industry recognition, expert quotes |
| Trustworthiness | Contact info, privacy policy, reviews, HTTPS, publication dates |

Word count: floors, not targets. Blog: 1,500+ for coverage. Homepage: 500+. Flesch score 60-70 for general audiences — not a direct ranking factor.

### AI Citation Readiness (AEO/GEO — 2025+ priority)
AI search (ChatGPT, Perplexity, Google AI Mode) = "zero blue links" means AI citation is the only visibility path.
Optimization signals:
- Clear, quotable statements (one claim per sentence)
- Answer-first structure (direct answer in first 50 words, explanation follows)
- Strong H2/H3 heading hierarchy
- Structured data (FAQ, HowTo, Article schema)
- Strong E-E-A-T signals (see above)

### Programmatic SEO Patterns (12 playbooks)
Best patterns for creator/SaaS context:
1. **Templates**: "[Type] template for [use case]" — high intent, long tail
2. **Comparisons**: "[Product A] vs [Product B]" — BOFU, high conversion
3. **Examples**: "Best [type] examples" — TOFU, high shareability
4. **Locations**: "[Service] in [City]" — local-adjacent use cases
5. **Integrations**: "[Product] + [Tool] integration" — captures tool-search traffic
6. **Glossary**: Define industry terms — builds authority, AI citation target

Data hierarchy: Proprietary > product-derived > user-generated > licensed > public. Strong data = strong pages.
URL structure: subfolders over subdomains (subfolders consolidate domain authority).
Quality over quantity: 100 great pages beats 10,000 thin ones — Google penalizes doorway pages.

## 10. Lead Magnets & Engineering-as-Marketing

### Lead Magnet Categories by Effort and Stage
| Type | Effort | Best Stage | Conversion benchmark |
|---|---|---|---|
| Checklist, cheat sheet, template | 1-8 hours | Awareness | 20-40% warm, 5-15% cold |
| Quiz, interactive tool | 1-2 weeks | Awareness/Consideration | Higher engagement, lower fill rate |
| Mini-course, webinar | 1-2 weeks | Consideration | Higher intent, filters audience |
| Ebook, video course | 1-3 weeks+ | All stages | Lower conversion, higher authority |
| Calculator, analyzer | 2-4 weeks | Consideration/Decision | High intent, strong product signal |

### Buyer Stage Matching
- Awareness → checklist, guide, short quiz
- Consideration → comparison, webinar, case study, assessment
- Decision → template, free trial, implementation guide, calculator

### Form Field Rule
Email only = highest conversion. Each additional field: -5 to -10% completion. Ask for minimum viable data.

### Free Tool Strategy (Engineering-as-Marketing)
Best tool categories: calculators, generators, analyzers, testers.
Strategic scoring (rate your concept 1-5 each):
- Search demand: are people already searching for this?
- Audience fit: does your ICP actually face this problem?
- Product adjacency: does using the tool create a natural path to your product?
- Build feasibility: can it be MVP'd in <2 weeks?
- Shareability: does the result beg to be shared?

Score >25/40 → strong candidate. Below 20 → reconsider.

Gating options: full gate (email required), partial gate (use free, save/export requires email), ungated (traffic + backlinks, no leads). Partial gating usually optimal for tools with export value.

### Content Repurposing Multiplier
One pillar (60-90 min video OR 2000+ words) → 10+ formats:
Video → Reels (4-6 clips) → Carousel (3-5) → Newsletter → Blog → Stories → LinkedIn → PDF checklist
Batch weekly: 300% more engagement at 60% less time vs. creating each format independently.

## 11. Decision Matrix — Pick the Right Framework

| Situation | Use |
|---|---|
| Writing a hero headline | Benefit-led + number (Section 5), emotional contrast hook |
| Writing cold email | Observation→Problem→Proof→Ask, 100 words max, signal-based |
| Qualifying a lead | BANT (fast) or MEDDPICC (enterprise) |
| High-ticket B2B meeting | SPIN (Situation→Problem→Implication→Need-Payoff) |
| Handling an objection | Validate→Isolate→Reframe + Voss labeling |
| Timing objection | "What changes in 90 days?" + quantify cost of delay |
| Writing a pricing page | 3-tier + anchoring + charm vs. rounded pricing decision |
| Choosing a content format | Match Schwartz awareness level to stage; cold = TOFU |
| Writing a landing page | AIDA + Hormozi value equation + Section 5 section order |
| Writing nurture email | PAS or BAB + one topic + one CTA, 150-300 words |
| Negotiating price | Ackerman model + no-oriented questions + accusation audit |
| Choosing a lead magnet | Match buyer stage table (Section 10) |
| SEO content brief | E-E-A-T + answer-first + heading hierarchy + schema |
| Programmatic SEO page | Pick playbook pattern (Section 9) + proprietary data |
| Ad creative test | 3-5 angle batches + platform char limits + 2s hook rule |
| Retargeting strategy | Retargeting ladder (Section 7) |
| Instagram post | `ig-content-creator` skill + hook angle competition |
